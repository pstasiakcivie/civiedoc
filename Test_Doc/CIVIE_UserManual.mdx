# Calendar

The **Calendar Tab** in the **CIVIE** application provides a **centralized scheduling system** for managing patient appointments, resource availability, and location-specific scheduling. It allows users to view, create, edit, and manage appointment slots, ensuring efficient workflow management for radiology and medical practices.

**Key Features of the Calendar Tab**

1. **Appointment Scheduling** ‚Äì Create, modify, reschedule, and cancel appointments.

2. **Resource & Location Management** ‚Äì Assign appointments to specific facilities, rooms, and modalities.

3. **Filters & Search** ‚Äì Quickly filter appointments by **location, resource, status, and captions**.

4. **View Options** ‚Äì Switch between **day, week, list, and status views** for better visibility.

5. **Blocked Slots Management** ‚Äì Track unavailable slots due to holidays, tech unavailability, or other constraints.

6. **Waitlist Management** ‚Äì Manage patient waitlists for available slots.

* **Appointment Booking**

* **Block Slot**

* **Running Late**

* **Show Waitlist**

* The **entire series** of a recurring blocked slot (e.g., weekly, monthly).

**Declining a Single Occurrence**

<Steps>
  <Step title="Right-click on the blocked slot." />

  <Step title="The selected occurrence is removed from the calendar." />

**Declining an Entire Series**

<Steps>
  <Step title="Right-click on the blocked slot." />

  <Step title="Select Decline from the menu." />

  <Step title="Choose Series to remove all occurrences of the blocked slot." />

  <Step title="Confirm the action by clicking Yes on the pop-up." />

  <Step title="The entire blocked slot series is deleted from the calendar." />
</Steps>

**Deleting a Blocked Slot**

If a **blocked slot is non-recurring (single occurrence)** and the user wants to remove it, follow these :

<Steps>
  <Step title="Right-click on the blocked slot." />

# Incoming Documents

The **Incoming Documents** tab in **CIVIE** is used to **manage, categorize, and process received documents** related to patients, orders, and billing. This module helps streamline document workflow and ensures that important medical records are efficiently handled.

**Key Features of the Incoming Documents Tab**

* **Document Summary Dashboard** ‚Äì Displays document counts for different time periods.

* **Worklists** ‚Äì Categorizes documents into **Personal** and **Shared Worklists** for tracking.

* **Document Upload** ‚Äì Allows users to manually upload patient-related files.

* **Filters & Search** ‚Äì Quickly locate documents based on time range and worklist type.

* **Integration with Orders & Patients** ‚Äì Link documents directly to patient records and orders.

| **Column Name**  | Description                                                        |
| ---------------- | ------------------------------------------------------------------ |
| Document #       | Unique identifier for each document.                               |
| File Name        | Name of the received or uploaded file.                             |
| Status           | Indicates if the document is **New, In Progress, or Completed**.   |
| Referring Doctor | Displays the name of the referring physician (if linked).          |
| Location         | Specifies the facility where the document was received.            |
| Patient          | Links the document to a specific patient (if assigned).            |
| Order            | Links the document to a medical order (if applicable).             |
| E-mail           | Displays the sender‚Äôs email if received via email.                 |
| Calling Number   | Shows the number associated with the document if received via fax. |
| Actions          | Options to **view, edit, or assign** the document.                 |
