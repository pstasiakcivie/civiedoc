## Creating a Coding Audit Worklist

<Steps>
  <Step title="Start Worklist Creation">
    Click the \+ icon in either **My Private Worklist** or **Shared Worklist**.

    The **Create Worklist** page opens.
  </Step>
  <Step title="Enter Worklist Details">
    **Worklist Name:** Enter a descriptive name (e.g., Audit Review).

    **Worklist Type:** Defaults to Coding Audit.

    **Description (Optional):** Add reference details.

    **Mark as Private:** Checked by default. Uncheck to create a shared worklist.

    **Allow Shared Users to Edit/Delete:** Enable for shared editing rights.
  </Step>
  <Step title="Add Users or User Groups">
    Select a group from the **User Group** dropdown.

    Users linked to the group auto-populate under **Select User**.

    To add more users, select them individually.

    If the group doesn't exist:
    Click **Create User Group** â†’ Enter Group Name â†’ Add Users â†’ Click **Save**.
  </Step>
  <Step title="Apply Filters to Select Claims">
    Click the \+ icon under **Filter Items**.

    Choose a **Field** (e.g., Claim Status), an **Operator** (e.g., Equals), and a **Value** (e.g., Pending Review).

    Use **AND/OR** to combine filters.
  </Step>
  <Step title="Preview and Save">
    Click **Preview** to view filtered claims.

    Click **Save** to create the worklist.

    <Note>
      _Note: Filters help refine claim selection for efficient audits._
    </Note>
  </Step>
  <Step title="Manage the Worklist">
    The new worklist appears under **My Private Worklists** or **Shared Worklists**.

    Open it to begin auditing coding claims.
  </Step>
</Steps>

## **Managing Coding Audits**

### **Starting a Worklist**

<Steps>
  <Step title="Start Worklist" icon="chevron-down">
    Select the desired worklist from **Personal**, **Shared**, or **All (Quick Access)**.

    Click **Start Worklist** next to it to begin auditing.
  </Step>
</Steps>

### **Previewing a Worklist**

<Steps>
  <Step title="Preview Worklist" icon="chevron-down">
    Click **Preview** to review all records in the selected worklist.

    View claims, patient records, and coding items in one place.
  </Step>
</Steps>

### **Reviewing Coding Audit Claims**

<Steps>
  <Step title="Access Worklist and Claim" icon="chevron-down">
    Go to the **Coding Worklist** section and choose a **Personal** or **Shared** list.

    Use **Search** and **Filter** options to find a specific claim.

    Click the **View** icon next to the claim to open the coding report.
  </Step>
</Steps>

## **Finalizing the Audit**

<Steps>
  <Step title="Verify and Accept" icon="chevron-down">
    Review coding details in the right panel.

    Click **Accept** if the report is correct.

    <Info>
      _Accepted claims are finalized and removed from the queue._
    </Info>
  </Step>
</Steps>
