# **Navigate to Order Entry**

<Steps>
  <Step title="To access the Order Entry interface:" icon="chevron-down">
    1. Navigate to the **Order Entry** tab in the left navigation menu.
    2. Use the **Search** bar to locate documents by file name.
    3. Click **Filter** to refine the results using one or more criteria:
       - Created Date Range
       - Client Name
       - File Name
       - Document Status
    4. Click **Export** to download the filtered list in CSV or Excel format.
    5. Click the **View** button \*\*(in the Actions column) \*\*to open a specific order entry record.
  </Step>
</Steps>

# View Order Entries

This section guides users through viewing, editing, and managing order entry records in the RCM application.

<Steps>
  <Step title="Navigate to Order Details">
    To view and manage an order entry:

    Click **View** in the **Actions** menu next to the desired entry.

    <img
      src="/o5.png"
      alt="+newappointment Pn"
      title="+newappointment Pn"
      style={{ width:"92%" }}
      className="mx-auto"
    />
  </Step>
  <Step title="The interface opens in a split view">
    **Left Panel**: Displays the uploaded order document (PDF).

    **Right Panel**: Lists associated patient forms.

    <img
      src="/o6.png"
      alt="+newappointment Pn"
      title="+newappointment Pn"
      style={{ width:"89%" }}
      className="mx-auto"
    />

    <Note>
      **Document Viewer Note**\
      If the PDF fails to load, an error message appears: **_An error occurred while loading the document. Try reloading the page._**
    </Note>
  </Step>
  <Step title="Check Document Status">
    The current document status appears at the top of the right panel and updates automatically as form progresses, the status can be as follow:

    | Status          | Description                             |
    | :-------------- | :-------------------------------------- |
    | **New**         | Document has not been opened or edited. |
    | **In Progress** | Partially completed.                    |
    | **Completed**   | All required fields are filled.         |
    | **Invalid**     | Document has been marked as incorrect.  |

    To manually update the status:

    Click the **Actions** button at the top right. Choose **Mark as Complete** or **Mark as Invalid** based on the document's state.
  </Step>
  <Step title="Add Comments (If any)">
    Click **Comments** to view or add remarks.

    <img
      src="/o7.png"
      alt="+newappointment Pn"
      title="+newappointment Pn"
      style={{ width:"81%" }}
      className="mx-auto"
    />

    Use the **Text Box andAttachment**  option to add new comments or upload related files.

    If there are no comments, the message **No comments found** will be displayed.
  </Step>
</Steps>

### **Add New Order (for Existing Patients)**

<Steps>
  <Step title="Add New Order">
    Click the **Add New Orde**r button.

    ![2025 07 23 17 38 40 Pn](/o9.png)
  </Step>
  <Step title="Search Patient">
    Search and select the **existing patient**.
  </Step>
  <Step title="Select Encounter">
    **Select an existing encounter** associated with the patient.\
    – If no relevant encounter exists, click the \*\*Add New Encounter \*\*button to create one.

    <img
      src="/o11.png"
      alt="2025 07 23 17 38 40 Pn"
      title="2025 07 23 17 38 40 Pn"
      className="mx-auto"
      style={{ width:"77%" }}
    />
  </Step>
  <Step title="Add New Encounter">
    If adding a new encounter enter the following details:

    **Start Date:** Start date of the encounter

    **End Date (Required):** Actual or expected end date

    **Patient Class:** Auto-filled for existing patients; otherwise select patient class

    **Attending Dr. NPI:** Enter National Provider Identifier

    **Attending Dr. Name:** Auto-populated after NPI entry

    <img
      src="/o12.png"
      alt="2025 07 23 17 38 40 Pn"
      title="2025 07 23 17 38 40 Pn"
      className="mx-auto"
      style={{ width:"81%" }}
    />

    <Note>
      **Note:**\
      Complete all required fields (marked with \*), including: Client Location, First Name, Last Name, Date of Birth, Gender, ZIP Code, MRN, SSN, Phone, Address, City, and State.
    </Note>
    Click **Save** to generate the encounter and the corressponding order.
  </Step>
  <Step title="Fill the Order Form">
    Fill in the **Order Form** with the required details.
  </Step>
  <Step title="Save">
    Click **Save** to submit the order.
  </Step>
</Steps>

## **Finalizing the Order Document**

- Once all patient orders are entered, click the **Actions** menu and select mark as **Complete** to update the status.
- If the document contains incorrect or invalid data, select mark as **Invalid.**
