# **Navigate to Order Entry**

<Steps>
  <Step title="To access the Order Entry interface:" icon="chevron-down">
    1. Navigate to the **Order Entry** tab in the left navigation menu.
    2. Use the **Search** bar to locate documents by file name.
    3. Click **Filter** to refine the results using one or more criteria:
       - Created Date Range
       - Client Name
       - File Name
       - Document Status
    4. Click **Export** to download the filtered list in CSV or Excel format.
    5. Click the **View** button \*\*(in the Actions column) \*\*to open a specific order entry record.
  </Step>
</Steps>

# View Order Entries

This section guides users through viewing, editing, and managing order entry records in the RCM application.

<Steps>
  <Step title="Navigate to Order Details">
    To view and manage an order entry:

    Click **View** in the **Actions** menu next to the desired entry.

    <img
      src="/o5.png"
      alt="+newappointment Pn"
      title="+newappointment Pn"
      style={{ width:"92%" }}
      className="mx-auto"
    />
  </Step>
  <Step title="The interface opens in a split view">
    **Left Panel**: Displays the uploaded order document (PDF).

    **Right Panel**: Lists associated patient forms.

    <img
      src="/o6.png"
      alt="+newappointment Pn"
      title="+newappointment Pn"
      style={{ width:"89%" }}
      className="mx-auto"
    />

    <Note>
      **Document Viewer Note**\
      If the PDF fails to load, an error message appears: **_An error occurred while loading the document. Try reloading the page._**
    </Note>
  </Step>
  <Step title="Check Document Status">
    The current document status appears at the top of the right panel and updates automatically as form progresses, the status can be as follow:

    | Status          | Description                             |
    | :-------------- | :-------------------------------------- |
    | **New**         | Document has not been opened or edited. |
    | **In Progress** | Partially completed.                    |
    | **Completed**   | All required fields are filled.         |
    | **Invalid**     | Document has been marked as incorrect.  |

    To manually update the status:

    Click the **Actions** button at the top right. Choose **Mark as Complete** or **Mark as Invalid** based on the document's state.
  </Step>
  <Step title="Mark as Invalid">
    The **Mark as Invalid** feature allows users to flag a order entry document as invalid if it cannot be used or processed. Here’s how you can complete this action:

    ![O19 Pn](/images/o19.png)

    1. **Reason**

       Select a predefined reason from the dropdown (e.g., _Can't Open_). This helps categorize the issue for reporting and resolution.
    2. **Comment**

       Optionally, add a comment to provide more context or specific details about why the document is invalid.
    3. **Mark as Invalid Button**

       Click **Mark as Invalid** to confirm and submit the action. The document will be flagged and removed from active processing.
    4. **Cancel Button**

       Use **Cancel** to discard changes and return without marking the file.

    <Note>
      **Note**: Once a document is marked as invalid, it will no longer appear in the active worklist for processing.
    </Note>
  </Step>
  <Step title="Add Comments (If any)">
    Click **Comments** to view or add remarks.

    <img
      src="/o7.png"
      alt="2025 07 31 16 33 49 Pn"
      title="2025 07 31 16 33 49 Pn"
      className="mx-auto"
      style={{ width:"73%" }}
    />

    Use the **Text Box andAttachment**  option to add new comments or upload related files.

    If there are no comments, the message **No comments found** will be displayed.
  </Step>
</Steps>

    14. **Reading Doctor** (Mandatory): Enter the name of the reading doctor (required)
    15. **Result Status**: Enter or select the result status
